Class warfare?

Nearly every grid system I've used since the invention of grid systems are class-based systems. They required some combonation of classes within the markup to achieve an infinite number of page layouts. This includes a project called CSS Smart Grid that Daniel Ryan wrote during the early days of the Obama campaign that we used for the next sixteen months. It was a lifesaver and much like Bootstrap it made developing rapidly possible. These systems changed the way most front-end developers outlined, prototyped, and eventually built their products, but they are not without their flaws:

  1. You need to include the entire grid (with all of it's columns and offset classes) whether I used them or not. This additional bloat in a gzipped CSS file is negliable, but it is still unnecessary in many scenarios.
  2. These systems added weight to the HTML document. Markup has changed since many of these systems were introduced and the documents themselves are increasing in size with the addition of microformats, aria roles, and ALL the meta data.
  3. Responsive flexibility starts to suffer. Recently, many of the projects I've worked on required different column layout across break points and this is difficult to achieve without using javascript or overriding the grid system.

These are all very solvable problems. The first iteration of Sagan was written approximately a year ago as nothing more than an idea for The New Yorker before I had officially started working as their Tech Lead. It made many of our responsive layouts much easier to build, provided more freedom to our designer, and resulted in measurable improvements in both performance and size of our code.

The Options

Sagan has a ton of options available so only generate the code you actually need such as standard offset option, ability to push or pull sections, opt outs for a helper .first class and/or :first-child psuedo selector, and a lot more. Additionally, initial flexbox support is available with much more support planned in future releases. Please check out the project on GitHub for a detailed list of the options and how to use them.

Installation

Sagan is available on GitHub or can be installed via Bower using the following command:

bower install sagan

Caveats

A system like Sagan is clearly not perfect for every scenario. Most prominently, layouts like this skeleton demo or this Smart Grid demo are much better achieved with a traditional class-based system because the additonal markup necessary for such layouts is much lighter than the additional css Sagan would generate.

Internet Explorer 8 and below support is disabled by default, but uncommenting a few lines within the code will fully support IE7/8 in all it's bloaty goodness. This support is dependent on the presence of an .oldie class on the html tag.

You can find flexbox support at Can I Use.
